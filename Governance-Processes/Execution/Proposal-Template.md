# Governance Proposal Template

---

> **How to use this template**
> Complete all sections marked **[Required]**. Sections marked **[If applicable]** are required only for certain proposal types as noted. Delete this instruction block before submitting.
>
> Before submitting, post a Draft on the governance forum for community discussion (minimum 5–7 days). Formal submission begins the Review Period (2–3 days) before voting opens (5–7 days).
>
> See the **Proposal & Voting Framework** for lifecycle details and the **DAO Parameters Registry** for current thresholds.

## 3. Category and Thresholds [Required]

Select the applicable category and confirm the required thresholds from the DAO Parameters Registry.

### 3.1 Category

- [ ] **Constitutional** — Changes to the Charter or core governance structure
  - Quorum: ≥ 10% of circulating XRD
  - Approval: ≥ 66% YES
- [ ] **Governance Process** — Changes to governance rules or process documents
  - Quorum: ≥ 7% of circulating XRD
  - Approval: ≥ 60% YES
- [ ] **Treasury & Budget** — Funding requests or budget allocations
  - Quorum: ≥ 5% of circulating XRD
  - Approval: ≥ 50% YES
- [ ] **Executable** — Technical or operational changes requiring on-chain or system execution
  - Quorum: ≥ 5% of circulating XRD
  - Approval: ≥ 50% YES
- [ ] **Signaling** — Non-binding community sentiment
  - Quorum: ≥ 3% of circulating XRD
  - Approval: ≥ 50% YES

### 3.2 Change Classification [If applicable — Governance Process and Constitutional proposals]

- [ ] **Patch** — Cosmetic or non-semantic change (typo corrections, formatting, cross-reference updates)
- [ ] **Minor** — Process improvement, parameter adjustment, or non-structural policy update
- [ ] **Major** — Change to Charter, governance structure, or any body's authority or rights
